## 압축 지시

이 지침을 다음 원칙에 따라 압축하라:

1. **의도 보존**: 각 지침의 근본 목적(WHY)은 반드시 유지
2. **고해상도 토큰화**:
   - 긴 설명 → 해당 도메인의 핵심 개념어/인물/프레임워크 참조로 대체
   - 예: 긴 협상 전략 설명 → "BATNA 원칙"
3. **암묵적 지식 레버리지**:
   - LLM이 이미 아는 것은 활성화 신호만 ("적용", "참조", "원칙에 따라")
   - 설명하지 말고 포인팅하라
4. **조건부 전개 구조**:
   - 세부사항은 "[조건] → [동작]" 형태로
   - "필요시 ~를 자체 생성" 패턴 활용
5. **중복 제거**: 다른 표현으로 반복되는 동일 의도 통합
